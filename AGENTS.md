# AGENT BOOTSTRAP

每次启动必须执行：

1. 读取当前目录下 `plan.md` 文件。无则创建一个。
2. 分析文件内容，梳理出接下来需要执行的任务。
3. 将当前需要执行的计划或者内容写入到 `plan.md` 中。
4. 执行了哪些、未执行哪些、待执行哪些都要及时同步信息到 `plan.md` 中。

## 执行规则补充

1. 调试或正式测试前，凡是涉及声卡播报、语音唤醒、全链路验证的任务，都必须先给设备上电，避免设备处于断电状态导致误判。
2. 长流程任务默认按阶段性上报结果，不要只在最后一次性汇总；至少应覆盖：
   - 已上电/前置准备完成
   - 音频预检或探针结果
   - `build` 结果
   - `burn` 结果
   - `run` 当前进度
   - 失败重跑/收尾状态
   - 最终汇总与归因结论
3. 如果发现问题，优先区分是设备/环境问题、用例问题、工具问题还是固件问题，并在 `plan.md` 中记录归因过程。
